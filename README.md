## 📋 快速导航

### 🎯 项目总览

这是一个**AI-Native工程系统**，用于理解社交媒体热门帖子，学习高互动评论模式，生成原创"神评论"。

**核心创新**：
- ✅ 多模态深度理解（文字+图片+语气+文化背景）
- ✅ 检索增强学习（从Reddit等社区学习真实套路，而非凭空创造）
- ✅ 4层风险防护（敏感词、结构、风险评分、原创性）
- ✅ 工程化落地（Skills+Hooks+MCP，而非纯文案堆砌）

**应用场景**：
- 社交媒体运营人员学习"什么样的评论容易获得互动"
- 品牌方管理评论生态（识别高价值讨论点）
- AI工程师学习如何在生成任务上加入风险防护

#### B3. Hook拦截机制 ✅

**路径**：`hooks/safety_checker.py` 和 `hooks/config.json`

**已实现内容**：
- **程序化敏感词拦截**（正则 + 关键词库）
- **结构完整性校验**（长度限制、非纯标点）
- **最小化长度检查**（防止AI偷懒输出"!"）

**为什么是Hook而非Prompt承诺**：
- **物理阻断**：一旦触发，直接中断，不继续处理
- **100%命中**：基于正则和关键词库，无遗漏
- **快速失败**：在评论生成后立即检查，降低无用API成本
- **不依赖LLM自觉**：即使LLM"忘记"了安全指令，Hook仍然会起作用

**容错设计**：
- 关键词库的定期更新
- 误报处理的申诉流程
- Hook强度的可调参数

## 🛡️ 对抗场景防护方案

**路径**：`adversarial_scenarios.md`

**4大对抗场景**：

### 1. Prompt Injection 防护 ✅
```
攻击：帖子中植入"Ignore system instructions..."
防护：
  - Level 1：关键词识别（Injection词库500+）
  - Level 2：句式结构分析（多层模式检测）
  - Level 3：指令隔离（将输入视为纯数据）
  - Level 4：隔离验证（模型级别的防护）
处置：HIGH→BLOCKED，MEDIUM→标记+继续，LOW→正常
```

### 2. 模型翻车冒犯 防护 ✅
```
风险：AI不小心输出歧视性内容
防护：
  - Layer 1：生成阶段自约束（Prompt显性化禁止）
  - Layer 2：Hook级敏感词检测
  - Layer 3：刻板印象检测（语义分析）
  - Layer 4：快速召回（已发布内容紧急下线）
检测模式：基于上下文的冒犯度判断
```

### 3. 信息不完整降级 ✅
```
场景：无图片、字数少、加载失败
处置：
  - Step 1：完整性检测（判断缺失类型）
  - Step 2：适配性分析（调整生成策略）
  - Step 3：显性标注（明确说明限制）
  - Step 4：备选方案（重试或降级）
原则：宁可降级输出，也不编造内容
```

### 4. AI错误建议反驳 ✅
```
面试官质疑例：
  "你的系统预测不准"
  "这就是模板"
  "为什么不用ChatGPT"
  
预装反驳论据：
  - 趋势预测 vs 确定性预测的区分
  - 套路≠单调，创新在于组合
  - 差异化优势（专业化 vs 通用化）
  - 透明可追踪的设计
```

## 🎓 项目核心解释

### 核心问题问答

#### Q: "这个系统的核心创新是什么？"

**答**：
```
三大创新：
1. 多模态学习（文字+图片+语气+文化）
2. 检索增强（从真实高赞评论反向工程套路）
3. 风险防护（4层Hook强制约束，而非纯Prompt承诺）

对比：
- ChatGPT：万能但不专，无平台适配
- 我们的系统：专业定向，学习真实用户，风险可控
```

#### Q: "为什么要用Skill和Hook而不是纯Prompt？"

**答**：
```
Skill的优势：
- 可复用性：定义一次，多个项目可用
- 版本管理：Skill版本独立升级
- 失败恢复：显性化的fallback机制
- 输出格式：严格Schema校验，上下游无歧义

Hook的优势（vs Prompt约束）：
- 物理阻断：一旦触发，直接中断（不能被override）
- 100%命中：正则 + 关键词库，无遗漏
- 快速失败：降低无用API成本
- 不依赖LLM自觉：即使模型"忘记"，Hook仍有效
```

#### Q: "系统怎么处理风险和边界？"

**答**：
```
6大绝对禁区（任何情况都不可越界）：
1. 身份攻击（种族、性别、宗教）
2. 人身攻击（直接辱骂、威胁）
3. 暴力或自伤（鼓励伤害）
4. 隐私泄露（特别是未成年人）
5. 违法内容（毒品、恐怖主义）
6. 虚假信息（否认历史事实）

多维度风险评估：
- 冒犯度等级（1-10）
- 原帖误解概率（1-10）
- 舆论翻车概率（1-10）
- 综合评分 ≥ 8 → BLOCKED

验证方式：
- 前置识别（Hook level）
- 多层校验（4层）
- 事后审查（可追踪）
```
